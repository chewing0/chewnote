# 日程界面优化方案

## 1. 背景与问题
当前日程页已具备“日历 + 全部日程 + 编辑删除”能力，但在高频使用场景下仍存在以下可用性问题：

- 日期切换成本高：主要依赖大日历控件，用户切换“今天/明天/后天”不够快。
- 信息层级弱：选中某天后，用户难以第一眼获得“当天有几项、是否有下一条安排”。
- 空状态缺少引导：某天无日程时缺少明确反馈与后续建议。
- 全部日程视图不够贴近习惯：默认按日期全量展示，用户常见诉求是“优先看未来安排”。

## 2. 常见用户习惯
参考日历和待办类产品的常见使用模式，用户通常会：

- 高频查看今天和明天，低频查看远期日期。
- 先看摘要（总数、下一件事）再看明细。
- 在“全部列表”里优先查看未来安排，而不是历史记录。
- 需要明确的空状态反馈，避免误以为数据加载失败。

## 3. 优化目标

- 将“今天/明天”切换操作控制在 1 次点击内。
- 在日历视图中强化“选中日期摘要 + 明细”结构。
- 为无日程场景提供明确空状态说明。
- 在全部日程中提供符合习惯的筛选（未来优先 / 全部 / 历史）。

## 4. 交互改造方案

### 4.1 日历视图

- 保留 DatePicker 作为完整日期选择器。
- 在 DatePicker 上方新增快捷日期操作：
  - 今天
  - 明天
  - 7天后
- 新增“选中日期信息卡”，展示：
  - 日期（yyyy年MM月dd日）
  - 星期
  - 当日安排数量
- 当天无安排时，展示空状态卡：
  - 明确提示“当天暂无安排”
  - 提示可在聊天页通过自然语言新增

### 4.2 全部日程视图

- 新增筛选 Chip：
  - 未来优先（默认）
  - 全部
  - 历史
- 未来优先：仅显示今天及未来日程，按日期时间升序。
- 历史：仅显示今天之前日程，按日期时间降序。
- 按日期分组展示，保留现有“编辑/删除”。

## 5. 视觉与文案策略

- 日期摘要卡使用弱强调背景，强化信息层级。
- 日期分组头保留简洁样式，避免抢占主内容注意力。
- 空状态使用中性语气，避免“报错感”文案。

## 6. 验收标准

- 可以一键切换到今天/明天/7天后。
- 选中日期后可立即看到“日期 + 星期 + 数量”。
- 当天无安排时出现明确空状态卡。
- 全部日程支持未来优先/全部/历史筛选，默认“未来优先”。
- 编译通过，既有编辑/删除流程不回归。
